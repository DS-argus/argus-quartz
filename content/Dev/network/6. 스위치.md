### 스위치 : 여러 호스트를 연결할 수 있는 데이터 링크 계층의 네트워크 장비
- 허브와의 차이점
	- MAC주소를 학습해 특정 MAC 주소를 가진 호스트에만 프레임 전달
	- 전이중 모드로 동작
- 따라서 포트별로 콜리전 도메인이 나뉘고, 전이중 모드로 통신하므로 CSMA/CD 프로토콜 필요 없음
##### 특징
- MAC 주소 학습 : 특정 포트와 해당 포트와 연결된 호스트의 MAC 주소와의 관계 기억
	- 프레임 내의 "[[3. 이더넷#^1ab93c|송신지 MAC 주소]]" 필드를 바탕으로 이루어짐
		- 프레임 수신하면, 송신지 MAC 주소 바탕으로 테이블에 저장
	- MAC 주소 테이블에 스위치의 포트와 연결된 호스트의 MAC 주소 연관 관계 저장  
		![[6. 스위치 - 2025-05-14 - 17-19-39.png|470x319]]

### MAC 주소 학습의 원리
- 스위치의 세 가지 기능을 통해 이루어짐
	- 플러딩 : 목적지 MAC 주소가 MAC 주소 테이블에 없을 때, 스위치는 해당 프레임을 모든 포트로 전송하는 동작
		- 초기 통신 시 자주 발생
	- 포워딩과 필터링
		- 필터링 : 전달받은 프레임을 어디로 내보내고 어디로 내보내지 않을지 결정하는 기능
		- 포워딩 : 프레임이 전송될 포트에 실제로 프레임을 내보내는 기능
	- 에이징 : MAC 주소 테이블의 항목이 **일정 시간 동안 사용되지 않으면 자동 삭제**하는 기능
		- 기본적으로 5분 정도
		- 필요한 이유
			- 포트 변경에 대응
			- MAC 테이블 크기 제한 (스위치의 메모리는 제한적)
			- 네트워크 안정성 확보
			- 동적 환경 대응
- 시나리오
	- 호스트 A, B, C, D가 각각 포트 1, 2, 3, 4에 연결됨
	- 처음에는 MAC 주소 학습 전이라서 연관 관계 모름
	- 호스트 A가 C로 신호 전송 => 송신지 MAC 주소 확인해서 테이블에 A 주소 추가
	- 호스트 C는 모르기 때문에 일단 모든 포트로 다 송신 <= 플러딩(flooding) 
		- 호스트 B, D는 폐기
		- 호스트 C는 스위치로 응답 프레임 전송 => 테이블에 C 주소 추가
	- 이제 A->C 혹은 C->A 는 플러딩 발생하지 않고 필터링 & 포워딩 발생
	- 테이블에 등록된 특정 포트에서 일정 시간 프레임 전송 받지 못하면 해당 항목 삭제 <= 에이징 (aging)

> [!info] 브리지 (bridge)
> - 스위치와 유사한 장비
> - 네트워크 영역을 구획하여 콜리전 도메인을 나누거나 네트워크 확장하는 용도로 사용
> 	- "허브 - 브리지 - 허브" 형태로 활용
> - 스위치 기능 제공
> 	- MAC 주소 학습
> 	- 포워딩, 필터링 가능
> - 하지만 최근 스위치가 브리지 기능 포괄하고 더 다양해서 사용 빈도 감소

### VLAN (Virtual LAN)
- 스위치의 또다른 중요한 기능
	- 메시지 주고받을 일이 적거나 [[1. 네트워크 거시적으로 살펴보기#^67116e|브로드캐스트]] 메시지를 받을 필요가 없어서 같은 LAN에 속할 필요가 없는 호스트가 있을 수 있음
	- 분리를 위해 매번 스위치 장비 구매하는 것은 낭비 => VLAN 구성으로 해결
- 한 대의 물리적 스위치로 _여러 대의 스위치가 있는 것_ 처럼 LAN 구획 가능  
	![[6. 스위치 - 2025-06-04 - 00-23-42.png|771x358]]
	- VLAN1에 속한 호스트들은 서로 동일한 LAN에 있는 것으로 인식
	- 하지만 VLAN2에 있는 호스트들은 물리적 거리에 관계없이 다른 LAN에 있는 것으로 인식
	- 서로 통신하고자 하면 네트워크 계층 이상의 상위 계층 장비 필요
##### 포트 기반 VLAN (port based VLAN) : 스위치의 포트가 VLAN 결정
- 특정 포트에 VLAN 할당 후, 해당 포트에 호스트 연결시켜서 VLAN 결정
- VLAN 트렁킹 (VLAN Trunking) : 스위치 간의 통신을 위한 '트렁크 포트'에 VLAN 스위치를 서로 연결해서 두 대 이상의 VLAN 스위치를 효율적으로 연결하여 확장
	- VLAN1 호스트 4개, VLAN2 호스트 3개, VLAN3 호스트 3개, 총 10개의 호스트를 포트가 8개인 스위치에 연결하는 것이 가능해짐  

![[6. 스위치 - 2025-06-04 - 00-25-47.png|784x439]]

> [!info] 트렁크 포트로 전달받은 프레임이 어떤 VLAN에 속하는지 파악하는 방법
> - 어떤 프레임이 스위치 A에서 트렁크 포트를 통해 스위치 B로 넘어왔다가 해보자
> 	- 만약 스위치가 아직 학습되지 않은 MAC 주소를 갖고 있다면 일반적인 이더넷 프레임으로 알 수 없음
> - 802.1Q 프레임 사용 : 어떤 VLAN에 속하는지 식별하기 위한 정보가 포함된 확장된 이더넷 프레임 사용
> 	- 헤더 송신지 MAC 주소와 타입/길이 사이에 32비트 크기의 VLAN 태그가 추가됨

##### MAC 기반 VLAN : 송수신하는 프레임 속 MAC 주소가 호스트가 속할 VLAN 결정
- 만약 호스트 A의 MAC 주소가 VLAN3에 할당되었으면, 호스트 A를 어떤 포트에 연결해도 호스트 A는 VLAN3에 속한 호스트
