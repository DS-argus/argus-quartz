### 이더넷 (Ethernet) : 다양한 통신 매체의 규격들과 송수신되는 [[2. 네트워크 미시적으로 살펴보기#^3d6ed0|프레임]]의 형태, 프레임을 주고받는 방법 등이 정의된 네트워크 기술
- 두 컴퓨터간 정보를 주고 받으려면 다음이 필요
	- [[1. 네트워크 거시적으로 살펴보기#^bf08b9|통신 매체]] (케이블)
	- 정보를 송수신하는 방법
- 물리 계층과 데이터 링크 계층이 밀접하게 연관된 이유 => '**이더넷**' 기술이 사용되기 때문
	- 물리 계층 : 이더넷 규격 케이블 사용
	- 데이터 링크 계층 : 주고받는 프레임은 이더넷 프레임 형식
- 현대 LAN, 특히 유선 LAN 환경에서 가장 대중적으로 사용되는 기술

---
### 이더넷 표준
- IEEE 802.3 : 전기전자공학자협회 (IEEE)가 이더넷 관련 기술을 표준화한 이름
	- 허브, 스위치, NIC (Network Interface Card), 케이블 등 대부분의 네트워크 장비들은 '특정 이더넷 표준'을 이해하고 따르고 있음
		- 특정 이더넷 표준을 지원한다...
		- 특정 이더넷 표준과 호환된다...
	- 이더넷 작업 그륩의 이름이기도 함
	- 이더넷 표준에 따라 지원되는 네트워크 장비, 통신 매체의 종류와 전송 속도가 달라짐

---
### 통신 매체 표기 형태 ([[2. 네트워크 미시적으로 살펴보기#^efa223|물리 계층]]과 관련)
- "전송 속도BASE-추가 특성" : 특정 이더넷 표준 규격에 따라 구현된 통신 매체를 지칭하는 방법
	- 100BASE-T : 100Mbps 속도를 지원하는 트위스티드 페이 케이블
	- 1000BASE-SX : 1Gbps 속도를 지원하는 단파장 광섬유 케이블
	- 5GBASE-T : 5Gbps 속도를 지원하는 트위스티드 페어 케이블
	- ....
- 전송 속도 (date rate)
	- 10 -> 10Mbps
	- 100 -> 100Mbps  (여기부터 고속 이더넷)
	- 1000 -> 1000Mbps = 1Gbps (기가비트 이더넷)
	- 10G -> 10Gbps = 10000Mbps (10기가비트 이더넷)
- BASE : BASEband의 약자로 '변조 타입' (modulation type)[^2] 의미
	- 일반적인 LAN 환경에서는 대부분 BASEband 방식 사용
	- 변조 타입의 종류
		- BASEband : 변조하지 않은 원래의 신호가 있는 주파수 대역. 가장 낮은 주파수부터 시작하며, 일반적으로 0Hz 근처의 신호를 포함
			- 주파수 변조 없이 디지털 신호 그대로 전송
			- 단거리 전송에 적합
			- 이더넷
		- BROADband : 넓은 주파수 대역을 사용하는 방식
			- 여러 주파수 채널을 동시에 사용해 여러 채널이나 신호를 동시에 전송가능
			- 주로 아날로그 변조 기반으로 여러 신호 병렬로 전송
			- 케이블 TV, 케이블 인터넷, 위성 통신
		- PASSband : 기저대역 신호를 고주파 신호로 변조하여 특정 주파수 대역만 통과시켜 전송하는 방식
			- 무선이나 장거리 통신에서 주로 사용
			- 변조가 필수적이고 고주파 캐리어 필요
			- 무선 LAN (Wi-Fi), 셀룰러 통신 (LTE, 5G), 라디오/TV 방송
- 추가 특성 : 통신 매체의 특성 명시
	- 전송 가능한 최대 거리 : 1000BASE-_2_, 1000BASE-_5_, ...
	- 물리 계층 인코딩 방식 : 1000BASE-C_X_, 1000BASE-T_X_, ...
	- 비트 신호 전송 가능한 레인 수 : 100GBASE-LR_4_
	- 통신 매체 종류 ^1f944a
		- C -> 동축 케이블
		- T -> 트위스티드 페어 케이블
		- S -> 단파장 광섬유 케이블
		- [[4. NIC와 케이블#코어 지름에 따른 구분|L]] -> 장파장 광섬유 케이블

- 이더넷 표준과 이를 기반으로 하는 통신 매체 일부 예시
	- IEEE 802.3i -> 10BASE-T
	- IEEE 802.3u -> 100BASE-TX
	- ...

### 이더넷 프레임 ([[2. 네트워크 미시적으로 살펴보기#^efa223|데이터 링크 계층]]과 관련)
- 이더넷 프레임은 [[2. 네트워크 미시적으로 살펴보기#^e384d8|캡슐화]] 과정을 통해 생성
	- _이더넷 프레임 헤더_ 의 구성 : 프리앰블 + 수신지 MAC 주소 + 송신지 MAC 주소 + 타입/길이
	- _이더넷 프레임 페이로드_ 의 구성 : 데이터
	- _이더넷 프레임 트레일러_ 의 구성 : FCS
-  헤더
	- 프리앰블 (preamble) : 이더넷 프레임 시작알리는 8byte 정보
		- 첫 7byte는 10101010, 마지막 byte는 10101011 : 수신지는 프리앰블로 이더넷 프레임이 오는 것을 식별
	- 수신지 / 송신지 MAC 주소 : 네트워크 인터페이스마다 부여되는 6byte 주소로 LAN 내의 수신지 / 송신지 특정 가능 ^1ab93c
		- MAC 주소는 일반적으로 고유하고 변경되지 않음
		- [[4. NIC와 케이블|NIC (network interface controller)]] 가 네트워크 인터페이스를 담당해서 한 컴퓨터에 NIC 여러개면 MAC 주소도 여러개 가능
		- `ipconfig` 로 확인 가능 -> ce:cd:fd:5e:53:a6 (16진수 12자리 -> 48bit -> 6byte)
	- 타입/길이
		- 타입 (이더타입, ethertype) : 이더넷 프레임이 '어떤 정보를 캡슐화했는지'를 나타내는 정보
			- 상위 계층에서 사용된 프로토콜 이름 명시
				- IPv4 프로토콜이 캡슐화된 정보 운반 -> 16진수 0800
				- IPv6 프로토콜이 캡슐화된 정보 운반 -> 16진수 86DD
				- ARP 프로토콜이 캡슐화된 정보 운반 -> 16진수 0806
- 데이터 : 네트워크 계층의 데이터 (IP 패킷) + 헤더 ^05fe1c
	- 상위 계층에서 전달받거나, 상위 계층오로 전달해야 할 내용
	- 최대 크기는 1500byte이고 최소 46byte 이상이어야 함 (부족하면 0으로 padding)
- FCS (frame check sequence) : 수신한 이더넷 프레임에 오류가 있는지 확인하기 위한 필드
	- 데이터 링크 계층에서 발생하는 오류 검출을 담당
	- CRC (cyclic redundancy check) : 순환 중복 검사. 오류 검출용 값
		- 송신지 : 프리앰블 제외한 나머지 필드 값으로 CRC 값 계산하고 FCS 필드에 명시
		- 수신지 : 수신한 프레임에서 프리앰블, FCS 제외한 나머지 필드 값을 바탕으로 CRC 값 계산하고 FCS 필드 값과 비교
		- 불일치 하면 해당 프레임 폐기

> [!note] 토큰 링 (Token Ring)
> - 이더넷 외의 또 다른 LAN 기술
> - 네트워크에서 호스트가 링 형태로 연결
> - 호스트끼리 돌아가며 토큰 주고받음
> - 토큰을 가지고 있어야 네트워크 내 다른 호스트에게 메시지 송신 가능

[^1]: 데이러 링크 계층에서 주고 받는 메시지 단위 (PDU)

[^2]: 비트 신호로 변환된 데이터를 통신 매체로 전송하는 방법
