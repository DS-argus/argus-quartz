### NIC (Network Interface Controller) : 호스트와 유무선 통신 매체를 연결하고 정보 변환을 담당하는 MAC 주소가 부여되는 네트워크 장비
- 호스트가 통신 매체를 통해 흐르는 전기, 빛 등 다양한 신호를 이해하려면 컴퓨터가 이해할 수 있는 정보 간의 변환이 필요
- NIC는 '귀', '입'에 비유 가능 -> '호스트를 네트워크 (LAN)에 연결하기 위한 하드웨어'

##### NIC 생김새
- NIC를 부르는 말은 다양함 
	- 네트워크 인터페이스 카드, 네트워크 어댑터, LAN 카드, 네트워크 카드, 이더넷 카드 등....
	- 초기에는 확장 카드 형태로 따로 연결해서 사용
- 최근에는 USB로 연결하거나, 마더보드[^1]에 내장된 NIC가 있음 
	- USB로 연결하는 형태  
		![[4. NIC와 케이블 - 2025-05-12 - 09-55-58.png]]
##### NIC 역할 : 통신 매체에 흐르는 신호를 호스트가 이해하는 프레임으로 변환하거나 반대로 호스트가 이해하는 프레임을 통신 매체에 흐르는 신호로 변환
- 호스트가 네트워크를 통해 송수신하는 정보는 NIC를 거침 -> 네트워크 인터페이스 역할 수행
- MAC 주소를 통해 자기 주소, 수신되는 프레임의 수신지 주소 인식
- 도달한 프레임이 자신과 무관한 수신지 MAC 주소가 명시된 프레임이라면 폐기 혹은 FCS 필드로 오류 검출해 폐기
- NIC마다 지원되는 속도도 다름. 고속 NIC 따로 있음  

> [!info] 네트워크 인터페이스 (Network Interface)
> 컴퓨터가 네트워크 (인터넷, 내부망 등)와 통신할 수 있게 해주는 "창구" 역할의 장치 도는 가상 장치
> - 종류 및 예시
> 	- 물리적 인터페이스 : NIC
> 		- 유선 랜카드 : eth0, en0 (케이블)
> 		- 무선 랜카드 : wlan0, en1 (Wi-Fi)
> 	- 가상 인터페이스 : 소프트웨어적으로 만들어진 것
> 		- 루프백 : lo, lo0 (자기 자신과 통신, `127.0.0.1)
> 		- 브릿지 : bridge0, docker0 (여러 네트워크를 묶어주는 가상 스위치)
> 		- 도커/가상머신용 : 컨테이너나 VM에서 만들어지는 네트워크 창구
> 
> 즉, 네트워크 인터페이스는 네트워크에 연결된 "포트 (문)" 또는 "입구"로 컴퓨터가 네트워크에 들어가거나, 다른 장치와 데이터를 주고받는 경로
> 
> NIC와 거의 유사한 의미이지만 엄밀히 말하면 NIC는 **실제 하드웨어** 를 말하고 네트워크 인터페이스는 **실제 하드웨어 + 가상 인터페이스** 를 의미

### 케이블 (Cable) : NIC에 연결되는 물리 계층의 유선 통신 매체
#### 트위스티드 페어 케이블 (Twisted Pair Cable)
- 호스트가 아무리 빨리 데이터 처리 가능해도, 통신 매체가 해당 속도 따라잡지 못하면 무용지물
- 트위스티드 페어 케이블은 가장 대중적인 LAN 케이블로 구리선으로 전기 주고받음
##### 생김새 : 케이블 본체 + 커넥터
- 커넥터 : 트위스티드 페어 케이블에서는 주로 RJ-45 사용  
	![[4. NIC와 케이블 - 2025-05-12 - 10-17-29.png]]
- 케이블 본체 : 구리 선이 두 가닥씩 꼬여 있음
	- 구리 선간 전자적 간섭으로 노이즈 발생이 쉬움 -> 그물 모양의 철사 또는 포일로 감싸 보호 (차폐)
		- 그물 모양의 철사 : braided shield
		- 그물 모양의 포일 : foil shield  
		![[4. NIC와 케이블 - 2025-05-12 - 10-18-44.png|535x215]][^2]
##### Shield에 따른 분류
- 크게 3개로 분류
	- STP (Shielded Twisted Pair) 케이블 : braided shield 사용
	- FTP (Foil Twisted Pair) 케이블 : foil shield 사용
	- UTP (Unshielded Twisted Pair) 케이블 : 아무것도 감싸지 않은 구리 선만 있음
- 더 세분화 -> "XX/YTP"
	- X : U, S, F 중 하나 혹은 두 개 명시 (UTP, STP, FTP)
	- Y : 꼬인 구리 선 쌍을 감싸는 실드 종류 명시
	- 예시
		- S/FTP : 케이블 외부는 braided shield로 보호, 각 구리 선 쌍은 foil shield
		- F/FTP : 케이블 외부, 구리 선 쌍 모두 foil shield
		- SF/FTP : 케이블 외부는 braided, foil shield, 구리 선 쌍은 foil shield
		- U/UTP : 아무것도 감싸지 않음
##### 카테고리에 따른 분류
- 카테고리 (Cat) : 트위스티드 페어 케이블의 성능 등급 구분하는 역할. 높을수록 좋음

| 특징       | Cat5       | Cat5e      | Cat6        | Cat6a     | Cat7      | Cat8      |
| -------- | ---------- | ---------- | ----------- | --------- | --------- | --------- |
| 지원 대역폭   | 100MHz     | 100MHz     | 250MHz      | 500MHz    | 600MHz    | 2GHz      |
| 주요 대응 규격 | 100BASE-TX | 1000BASE-T | 1000BASE-TX | 10GBASE-T | 10GBASE-T | 40GBASE-T |
| 전송 속도    | 100Mbps    | 1Gbps      | 1Gbps       | 10Gbps    | 10Gbps    | 40Gbps    |
#### 광섬유 케이블 (Fiber Optic Cable)
- 빛 신호 이용해 전송
	- 속도 빠름, 먼 거리 전송  가능, 노이즈 영향 감소 => 대륙 간 네트워크 연결에도 사용
##### 생김새 : 케이블 본체 + 커넥터
- 커넥터 : 종류가 다양. 주로 다음 LC, SC, FC, ST 4종류 사용  
	![[4. NIC와 케이블 - 2025-05-12 - 10-37-27.png|470x322]]
- 케이블 본체 : 빛을 운반하는 광섬유 다발로 구성
	- 코어 (core) : 광섬유의 중심으로 실질적으로 빛이 흐르는 부분
	- 클래딩 (cladding) : 코어를 둘러싼 부분으로 빛이 코어 안에서만 흐르도록 빛을 가두는 역할 (빛 굴절율 차이로)
##### 코어 지름에 따른 구분
- 싱글 모드 광섬유 케이블 (SMF) : 코어 지름 8~10$\mu m$ 
	- 지름이 작아 빛의 이동 경로가 하나 이상을  갖기 어려워서 single mode
	- 신호 손실 적어 장거리 전송에 적합 -> 수십 KM
	- 비용 높음
	- 장파장 빛 사용
	- [[3. 이더넷#^1f944a|대표적인 케이블 규격]] : 1000BASE-LX, 10GBASE-LR
- 멀티 모드 광섬유 케이블 (MMF) : 코어 지름 50~62.5 $\mu m$
	- 빛이 여러 경로로 이동 가능
	- 신호 손실 클 수 있어 장거리 부적합 -> 수백 M~ 수 KM 정도
	- 단파장 빛 사용
	- 대표적인 케이블 규격 : 1000BASE-SX, 10GBASE-SR





[^1]: CPU, 메모리, 보조기억장치 등 컴퓨터 주요 부품 연결할 수 있는 기판

[^2]: 우측 그림은 엄밀하게 S/FTP 케이블
