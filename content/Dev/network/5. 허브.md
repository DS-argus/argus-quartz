### 주소 개념이 없는 물리 계층

- 물리 계층에는 주소 개념 없음 -> 단지 호스트와 통신 매체 간의 연결 및 통신 매체상의 송수신만 발생
- 데이터 링크 계층에는 주소 개념 있음 -> MAC 주소. 따라서 데이터 링크 이상 계층의 장비들은 송수신지 특정 가능하고 주소 바탕으로 정보에 대한 조작 및 판단 가능

### 허브 (Hub) : 여러 대의 호스트를 연결할 수 있는 물리 계층의 장비
- 이더넷 네트워크의 허브는 이더넷 허브라고 하며 커넥터를 연결할 수 있는 포트 (port)가 있음  
	![[5. 허브 - 2025-05-14 - 16-47-29.png|470x313]]
- 요즘 인터넷 환경에서는 잘 쓰이지 않으나 특징은 알아둬야 함
##### 특징
1. 전달받은 신호를 다른 모든 포트로 그대로 다시 내보냄
	- 물리 계층 장비인 허브는 주소 개념 없어 수신지 특정 못함
	- 따라서 신호 전달 받으면 _송신지를 제외한 모든 포트에 그저 내보내기만 함_
	- 신호 전달 받은 모든 호스트는 데이터 링크 계층에서 패킷의 MAC 주소 확인하고 관련 없는 주소는 폐기
2. 반이중 모드 (half duplex)로 동작
	- 1차선 도로처럼 송수신을 번갈아 가면서 하는 통신 방식
		- 동시에 송수신 불가능 (무전기)
		- 전이중 모드 (full duplex) : 동시에 양방향 송수신 가능

> [!info] 리피터 (Repeater)
> - 허브 이외의 물리 계층의 대표적인 장비
> - 전송 거리에 따른 신호 감소 혹은 왜곡 방지를 위해 전기 신호 증폭하는 장비로 마찬가지로 _그저 증폭시키기만 함_
> - 보통 허브가 리피터 기능을 포함

##### 콜리전[^1] 도메인 : 충돌이 발생할 수 있는 영역
- 충돌의 근본 원인 : 반이중 모드 통신
- 동일한 허브에 연결된 모든 호스트는 같은 콜리전 도메인에 속함
- 해결 방법
	- CSMA/CD 프로토콜 사용
	- 스위치 장비 사용

### CSMA/CD[^2] : 반이중 이더넷 네트워크에서 충돌을 방지하는 대표적인 프로토콜
- 유선 이더넷에서 사용하는 매체 접근 제어 (MAC)[^3] 방식
- 모든 장치가 전송 매체를 공유하는 네트워크에서, 다른 장치가 통신 중인지 확인 (Carrier Sense)한 후, 여러 장치가 동시에 전송하여 발생할 수 있는 충돌 (Collision)을 탐지(Detection)하고, 이를 해결하기 위한 재전송 절차를 포함하는 프로토콜
	- 사용환경 : 전통적인 유선 이더넷
	- 현재는 사용하지 않음 : 스위치 기반 이더넷에서는 각 포트가 전이중 모드로 동작하여 충돌 발생 안함
##### 작동원리
1. CS (Carrier Sense) : 캐리어 감지
	- 메시지 보내기 전에 네트워크 상에서 전송 중인 것이 있는지 확인
2. MA (Multiple Access) : 다중 접근
	- 복수의 호스트가 네트워크에 접근하려는 상황을 말하며 이 경우 충돌 발생
3. CD (Collision Detection) : 충돌 검출
	- 충돌 감지 시 전송 중단
	- 충돌 검출한 호스트는 충돌 발생을 알리는 잼 신호 (jam signal) 보내고 임의의 시간 동안 대기 후 다시 전송
	- 대기 시간은 랜덤하게 설정되며, 충돌 횟수 많아질수록 대기 시간 증가

> [!warning] 일상에서 주로 마주치는 허브는?
> 여러 장비를 연결해주는 멀티탭 느낌 -> USB 허브, 전원 허브 등등
> - 사용 목적 : 노트북이나 PC에 USB 포트를 확장하거나 전원/영상/기타 연결 통합
> - 방식 : 데이터/전원/영상 등 다양한 신호를 확장/분배
> - 계층 : OSI 계층과는 무관하거나 물리적 허브 수준
> - **네트워크 허브와는 전혀 관련 없음!**



[^1]: 허브에 동시에 신호를 송신하는 경우 발생
[^2]: Carrier Sense Multiple Access with Collision Detection
[^3]: Media Access Control. 여러 호스트가 공유하는 통신 매체(media)에 접근(access)할 때, 사용 가능한 제어(control) 방식
