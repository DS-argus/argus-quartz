### Signal : 프로세스에 보내는 소프트웨어 인터럽트
- 커널 또는 다른 프로세스가 특정 프로세스에게 **이벤트를 알리기 위해** 보내는 비동기적 통지
- 프로세스는 시그널을 받으면 기본 동작(종료, 무시, 코어 덤프 등)을 수행하거나 핸들러를 등록해 직접 처리할 수 있음
- 시그널 번호는 OS마다 약간 다를 수 있지만, 아래는 Linux 기준

##### 전체 시그널 목록 (번호순)

|  번호   | 이름                | 기본 동작      | 설명                            |
| :---: | :---------------- | :--------- | :---------------------------- |
|   1   | SIGHUP            | 종료         | 터미널 연결 끊김 / 데몬 설정 리로드         |
|   2   | SIGINT            | 종료         | 인터럽트 (`Ctrl+C`)               |
|   3   | SIGQUIT           | 종료 + 코어 덤프 | 종료 + 코어 덤프 (`Ctrl+\`)         |
|   4   | SIGILL            | 종료 + 코어 덤프 | 잘못된 명령어 (Illegal Instruction) |
|   5   | SIGTRAP           | 종료 + 코어 덤프 | 디버거 트랩 (breakpoint)           |
|   6   | SIGABRT           | 종료 + 코어 덤프 | 비정상 종료 (`abort()`)            |
|   7   | SIGBUS            | 종료 + 코어 덤프 | 버스 오류 (잘못된 메모리 정렬)            |
|   8   | SIGFPE            | 종료 + 코어 덤프 | 부동소수점 예외 (0으로 나누기 등)          |
|   9   | SIGKILL           | 즉시 종료      | 강제 종료 (**핸들링/무시 불가**)         |
|  10   | SIGUSR1           | 종료         | 사용자 정의 시그널 1                  |
|  11   | SIGSEGV           | 종료 + 코어 덤프 | 세그멘테이션 폴트 (잘못된 메모리 접근)        |
|  12   | SIGUSR2           | 종료         | 사용자 정의 시그널 2                  |
|  13   | SIGPIPE           | 종료         | 읽는 쪽이 닫힌 파이프에 쓰기 시도           |
|  14   | SIGALRM           | 종료         | 알람 타이머 만료 (`alarm()`)         |
|  15   | SIGTERM           | 종료         | 정상 종료 요청 (`kill` 기본값)         |
|  16   | SIGSTKFLT         | 종료         | 스택 폴트 (현재는 거의 사용되지 않음)        |
|  17   | SIGCHLD           | 무시         | 자식 프로세스 종료/정지 시 부모에게 전달       |
|  18   | SIGCONT           | 재개         | 정지된 프로세스 재개                   |
|  19   | SIGSTOP           | 일시정지       | 프로세스 강제 정지 (**핸들링/무시 불가**)    |
|  20   | SIGTSTP           | 일시정지       | 터미널 정지 (`Ctrl+Z`)             |
|  21   | SIGTTIN           | 일시정지       | 백그라운드 프로세스가 터미널 입력 시도         |
|  22   | SIGTTOU           | 일시정지       | 백그라운드 프로세스가 터미널 출력 시도         |
|  23   | SIGURG            | 무시         | 소켓에 긴급(Out-of-band) 데이터 도착    |
|  24   | SIGXCPU           | 종료 + 코어 덤프 | CPU 시간 제한 초과                  |
|  25   | SIGXFSZ           | 종료 + 코어 덤프 | 파일 크기 제한 초과                   |
|  26   | SIGVTALRM         | 종료         | 가상 타이머 만료                     |
|  27   | SIGPROF           | 종료         | 프로파일링 타이머 만료                  |
|  28   | SIGWINCH          | 무시         | 터미널 창 크기 변경                   |
|  29   | SIGIO             | 종료         | I/O 가능 알림 (비동기 I/O)           |
|  30   | SIGPWR            | 종료         | 전원 장애                         |
|  31   | SIGSYS            | 종료 + 코어 덤프 | 잘못된 시스템 콜                     |
| 34–64 | SIGRTMIN~SIGRTMAX | 종료         | 실시간 시그널 (사용자/앱이 자유롭게 정의)      |

##### 시그널 보내는 방법
```bash title="시그널 전송"
# kill 명령어 (이름과 달리 '죽이기'만 하는 게 아니라 시그널을 보내는 명령어)
kill -<시그널번호 또는 이름> <PID>

# 예시
kill -15 1234       # SIGTERM (번호)
kill -SIGTERM 1234  # SIGTERM (이름)
kill -TERM 1234     # SIG 접두어 생략 가능

# 프로세스 이름으로 보내기
killall -TERM python
pkill -TERM -f "etl_pipeline.py"

# 전체 시그널 목록 확인
kill -l
```

### 주요 시그널 정리

#### 프로세스 종료 관련

| 번호 | 이름 | 기본 동작 | 설명 | 언제 쓰는가 |
| :---: | :--- | :--- | :--- | :--- |
| 15 | **SIGTERM** | 종료 | 정상 종료 요청. 프로세스가 정리 작업(cleanup) 후 종료할 수 있음 | `kill` 명령의 기본값. **프로세스를 종료하고 싶을 때 가장 먼저 시도** |
| 9 | **SIGKILL** | 즉시 종료 | 강제 종료. 프로세스가 무시하거나 핸들링할 수 없음 | SIGTERM으로 안 죽을 때 최후의 수단. 정리 작업 없이 즉시 종료됨 |
| 2 | **SIGINT** | 종료 | 인터럽트. 터미널에서 `Ctrl+C` 입력 시 발생 | 포그라운드 프로세스를 즉시 중단하고 싶을 때 |
| 3 | **SIGQUIT** | 종료 + 코어 덤프 | 터미널에서 `Ctrl+\` 입력 시 발생 | 프로세스를 종료하면서 디버깅용 코어 덤프 파일을 남기고 싶을 때 |
| 6 | **SIGABRT** | 종료 + 코어 덤프 | 프로세스 스스로 비정상 상태를 감지했을 때 발생 | 직접 보내는 경우는 드묾. `abort()` 함수가 내부적으로 사용 |

```bash title="종료 시그널 실무 패턴"
# 1단계: 정상 종료 요청
kill 1234           # SIGTERM (기본값)

# 2단계: 몇 초 기다려도 안 죽으면 강제 종료
sleep 5 && kill -9 1234    # SIGKILL

# 실무에서 자주 쓰는 패턴 (스크립트)
kill $PID
timeout 10 tail --pid=$PID -f /dev/null  # 최대 10초 대기
if kill -0 $PID 2>/dev/null; then        # 아직 살아있으면
    kill -9 $PID                          # 강제 종료
fi
```

> [!warning] SIGKILL(9)을 바로 쓰지 말 것
> - SIGKILL은 프로세스에게 정리할 기회를 주지 않음
> - 임시 파일 미삭제, DB 커넥션 미반환, 파일 손상 등의 문제 발생 가능
> - 반드시 **SIGTERM → 대기 → SIGKILL** 순서로 사용

#### 프로세스 일시정지 / 재개 관련

| 번호 | 이름 | 기본 동작 | 설명 | 언제 쓰는가 |
| :---: | :--- | :--- | :--- | :--- |
| 19 | **SIGSTOP** | 일시정지 | 프로세스를 멈춤. 무시/핸들링 불가 (SIGKILL처럼 강제) | 프로세스를 잠시 얼리고 싶을 때 |
| 20 | **SIGTSTP** | 일시정지 | 터미널에서 `Ctrl+Z` 입력 시 발생. 핸들링 가능 | 실행 중인 작업을 백그라운드로 보내고 싶을 때 |
| 18 | **SIGCONT** | 재개 | 정지된 프로세스를 다시 실행 | SIGSTOP/SIGTSTP로 멈춘 프로세스를 다시 돌릴 때 |

```bash title="일시정지/재개 실무 활용"
# 시나리오: CPU를 많이 쓰는 배치 작업이 운영 서비스에 영향을 줄 때

# 배치 작업 잠시 멈추기
kill -STOP 8823

# 운영 서비스가 안정되면 다시 재개
kill -CONT 8823

# Ctrl+Z로 멈춘 프로세스 다루기
./long_job.sh        # 실행 중에 Ctrl+Z → [1]+  Stopped
bg                   # 백그라운드에서 계속 실행
fg                   # 다시 포그라운드로 가져오기
```

#### 설정 리로드 / 상태 관련

| 번호 | 이름 | 기본 동작 | 설명 | 언제 쓰는가 |
| :---: | :--- | :--- | :--- | :--- |
| 1 | **SIGHUP** | 종료 | 원래는 터미널 연결 끊김 알림. 많은 데몬이 **설정 파일 리로드** 용도로 사용 | 서비스 재시작 없이 설정만 다시 읽게 하고 싶을 때 |
| 10 | **SIGUSR1** | 종료 | 사용자 정의 시그널 1. 프로그램이 자유롭게 용도 정의 | 프로그램별 커스텀 동작 트리거 (로그 레벨 변경, 통계 출력 등) |
| 12 | **SIGUSR2** | 종료 | 사용자 정의 시그널 2 | SIGUSR1과 동일하게 프로그램별 커스텀 용도 |

```bash title="SIGHUP 실무 활용 — 설정 리로드"
# Nginx 설정 변경 후 재시작 없이 리로드
sudo kill -HUP $(cat /var/run/nginx.pid)
# 또는
sudo nginx -s reload   # 내부적으로 SIGHUP 전송

# sshd 설정 변경 후 리로드
sudo kill -HUP $(pgrep sshd)

# logrotate 후 로그 파일 다시 열기
kill -HUP $(pgrep rsyslog)
```

```bash title="SIGUSR1/2 실무 활용"
# dd 작업 진행 상황 확인 (dd는 SIGUSR1을 받으면 현재 진행률 출력)
dd if=/dev/sda of=/backup/disk.img bs=4M &
kill -USR1 $(pgrep dd)
# 1048576 bytes (1.0 MB) copied, 2.3 s, 456 kB/s

# Node.js 디버거 활성화
kill -USR1 $(pgrep node)
```

> [!tip] SIGHUP과 nohup의 관계
> - 터미널을 닫으면 해당 터미널에서 실행된 프로세스에 SIGHUP이 전송되어 종료됨
> - `nohup` 명령어는 SIGHUP을 **무시**하게 만들어 터미널을 닫아도 프로세스가 계속 실행됨
> ```bash
> nohup python long_job.py &    # 터미널 닫아도 계속 실행
> ```

#### 기타 알아두면 좋은 시그널

| 번호  | 이름           | 기본 동작      | 설명                             | 언제 쓰는가                                |
| :-: | :----------- | :--------- | :----------------------------- | :------------------------------------ |
| 13  | **SIGPIPE**  | 종료         | 읽는 쪽이 닫힌 파이프에 쓰려 할 때 발생        | 직접 보내기보다는 파이프라인 디버깅 시 이해 필요           |
| 14  | **SIGALRM**  | 종료         | `alarm()` 타이머 만료 시 발생          | 타임아웃 구현에 사용                           |
| 17  | **SIGCHLD**  | 무시         | 자식 프로세스가 종료되었을 때 부모에게 전달       | 좀비 프로세스 이해에 필요. 부모가 이걸 처리해야 자식이 정리됨   |
| 11  | **SIGSEGV**  | 종료 + 코어 덤프 | 잘못된 메모리 접근(Segmentation Fault) | 직접 보내지 않음. 프로그램 버그로 발생하며 디버깅 대상       |
| 28  | **SIGWINCH** | 무시         | 터미널 창 크기 변경 시 발생               | 직접 보내지 않음. `top`, `vim` 등이 화면 재조정에 사용 |
|  0  | **(시그널 0)**  | —          | 실제 시그널이 아님. 프로세스 존재 여부만 확인     | 프로세스가 살아있는지 체크할 때                     |

```bash title="시그널 0 — 프로세스 생존 확인"
# PID 1234가 살아있는지 확인 (시그널을 실제로 보내지 않음)
if kill -0 1234 2>/dev/null; then
    echo "프로세스 살아있음"
else
    echo "프로세스 없음"
fi
```

### 전체 요약 : 자주 쓰는 시그널 치트시트

```text
Ctrl+C  → SIGINT(2)    → "그만해"
Ctrl+Z  → SIGTSTP(20)  → "잠깐 멈춰"
Ctrl+\  → SIGQUIT(3)   → "코어 덤프 남기고 종료"

kill PID      → SIGTERM(15) → "정리하고 종료해줘" (기본값)
kill -9 PID   → SIGKILL(9)  → "지금 당장 죽어" (최후의 수단)
kill -HUP PID → SIGHUP(1)   → "설정 다시 읽어"
kill -STOP PID → SIGSTOP(19) → "얼려!"
kill -CONT PID → SIGCONT(18) → "다시 움직여"
kill -0 PID   → (없음)       → "살아있어?"
```

### 핸들링 가능 여부

| 시그널          | 무시 가능 | 핸들러 등록 가능 | 비고                  |
| :----------- | :---: | :-------: | :------------------ |
| SIGKILL (9)  |   ❌   |     ❌     | 무조건 즉시 종료           |
| SIGSTOP (19) |   ❌   |     ❌     | 무조건 일시정지            |
| 나머지 전부       |   ✅   |     ✅     | 프로그램이 자유롭게 동작 정의 가능 |

- SIGKILL과 SIGSTOP만 예외적으로 프로세스가 절대 가로채거나 무시할 수 없음
- 그래서 SIGKILL이 "최후의 수단"이 되는 것
