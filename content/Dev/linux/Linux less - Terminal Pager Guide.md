> [!abstract]+ TL;DR
> - `less`는 긴 출력을 한 화면씩 끊어서 보여주는 페이저 — `cat`과 달리 멈춰 세우고 위아래로 탐색 가능
> - 기본 사용은 `less 파일` 또는 `명령 | less`, 종료는 `q`, 검색은 `/패턴` 후 `n`/`N`
> - 자주 쓰는 옵션은 `-N`(줄번호), `-S`(긴 줄 자르기), `+F`(따라가기), `-i`(대소문자 무시)
> - `git log`·`man`이 내부적으로 `$PAGER`(보통 less)를 호출하는 구조

### 1. less란 무엇인가

터미널에서 긴 파일을 `cat`으로 열면 내용이 화면을 위로 주르륵 지나가 버린다. 마지막 화면만 남고 앞부분은 스크롤백을 뒤져야 한다. 로그나 긴 설정 파일을 읽을 때 불편한 지점이다.

**페이저**(pager)는 이 문제를 푸는 도구다. 긴 출력을 한 화면 단위로 끊어서 보여주고, 키를 눌러야 다음으로 넘어간다. 출력을 붙잡아두고 위아래로 오가며 검색까지 할 수 있다. `less`는 리눅스/macOS의 사실상 표준 페이저다.

> [!note]+ 이름의 유래 - "less is more"
> `less`라는 이름은 말장난이다. 선배 페이저가 `more`였고, 그 후속으로 더 강력한 페이저를 만들면서 영어 속담 "less is more"(적을수록 좋다)를 비틀어 `less`라고 붙였다. 이름은 "less"지만 기능은 more보다 훨씬 많다는 게 농담의 핵심이다.

### 2. 기본 사용법과 조작

가장 단순한 두 형태만 알면 시작할 수 있다. 파일을 직접 열거나, 다른 명령의 출력을 파이프로 넘긴다.

```bash
less /var/log/system.log   # 파일을 직접 연다
ps aux | less              # 다른 명령의 출력을 넘긴다
```

파일을 열면 첫 화면이 표시되고 `q`로 빠져나온다. 빠져나오면 화면이 less에 들어가기 전 상태로 깔끔하게 돌아가고, 출력이 스크롤백에 남지 않는다. 출력이 길 것 같으면 `| less`를 붙이는 습관이 유용하다.

이동과 검색은 키 하나에 즉시 반응한다. Enter 없이 `j`를 누르는 순간 한 줄 내려간다.

| 키 | 동작 |
|---|---|
| `j` / `k` | 한 줄 아래 / 위 |
| `Space` / `b` | 한 페이지 아래 / 위 |
| `g` / `G` | 파일 맨 처음 / 맨 끝으로 |
| `/패턴` / `?패턴` | 아래 / 위 방향 검색 |
| `n` / `N` | 다음 / 이전 검색 결과 |
| `q` | 종료 |

`vim`을 써봤다면 `j`/`k`/`g`/`G`/`/` 키가 똑같이 동작해 익숙할 것이다. 검색은 `/error` 입력 후 Enter를 치면 처음 나오는 매치로 이동하고, `n`/`N`으로 결과를 오간다.

옵션은 명령줄에서 `less -N 파일`처럼 주거나, less 안에서 입력해 토글한다. 자주 쓰는 네 가지만 추리면 이렇다.

```bash
less -N app.py              # 줄 번호 표시
less -S wide_table.csv      # 긴 줄을 접지 않고 잘라서 표시 (→ 키로 좌우 스크롤)
less +F /var/log/system.log # tail -f처럼 파일 끝을 실시간으로 따라간다
less -i config.yaml         # 검색 시 대소문자 무시 (대문자 섞이면 자동 구분)
```

- `-N`: 각 줄 앞에 줄 번호를 붙인다
- `-S`: 화면보다 긴 줄을 접지 않고 잘라낸다. CSV나 넓은 표가 뒤엉키지 않는다
- `+F`: 파일 끝으로 가서 새로 추가되는 내용을 따라간다. `Ctrl+C`로 멈추면 그 자리에서 스크롤·검색할 수 있고, `F`로 다시 따라간다
- `-i`: 검색어가 전부 소문자면 대소문자를 무시하고, 대문자가 섞이면 구분한다 (smart case)

### 3. cat, more와의 차이

`cat`은 파일 내용을 한 번에 전부 쏟아내고 끝나는 도구다. 페이징이 없다.

```bash
cat huge.log     # 전부 출력하고 종료 — 마지막 화면만 남음
less huge.log    # 첫 화면에서 멈춤 — 위아래로 탐색 가능
```

핵심 차이는 "멈추느냐"다. 긴 출력을 **읽는 게 목적**이면 less, 내용을 **다른 곳으로 흘려보내는 게 목적**이면 cat이 맞다.

`more`는 less보다 오래된 페이저다. 둘 다 페이징을 하지만 less가 기능이 훨씬 많다.

| | more | less |
|---|---|---|
| 위로 스크롤 | 제한적/불가 | 자유롭게 (`k`, `b`) |
| 검색 | 앞으로만 | 양방향 (`/`, `?`) |
| 끝까지 가면 | 자동 종료 | 화면 유지 |

오늘날 굳이 more를 쓸 이유는 거의 없다.

### 4. git, man이 less를 페이저로 쓴다

less를 직접 호출하지 않아도 우리는 이미 매일 less를 쓰고 있다. 출력이 긴 명령들이 내부적으로 페이저를 띄우기 때문이다.

- `git log`, `git diff`, `git show`: 결과가 길면 페이저로 보낸다
- `man`: 매뉴얼 페이지 전체를 페이저로 표시한다

이 페이저가 무엇인지는 `$PAGER` 환경변수가 결정하고, 설정이 없으면 대부분 less가 기본값이다. 그래서 less의 `j`/`k`/`/`/`q` 키가 git이나 man 화면에서도 똑같이 먹힌다.

```bash
echo $PAGER            # 현재 페이저 확인

export PAGER='less'
export LESS='-R -i'    # 색 코드 유지(-R), 검색 대소문자 무시(-i)
```

매번 옵션을 칠 필요 없이 `LESS` 환경변수에 넣어두면 less가 실행될 때마다 자동 적용된다. `git`은 별도로 `core.pager`도 보며, `git --no-pager log`로 페이저를 끌 수 있다.
